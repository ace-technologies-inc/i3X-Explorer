# Web Deployment Guide

This guide covers deploying i3X Explorer as a web application served by nginx. The web build is a standard static React app — there is no Node.js process to keep running after the build completes.

## Prerequisites

- Linux server with nginx installed
- Node.js at `/usr/bin/node` (v18+)
- A running I3X API server (the web app proxies to it via nginx)

## Overview

```
Browser → nginx :8090 → dist-web/        (static React app)
                  └─→ /api/ → I3X API server :8080
```

nginx serves the built static files and proxies `/api/` requests to your I3X API server. This means the browser only ever talks to one origin, so no CORS configuration is needed on the API server.

## First-Time Setup

### 1. Clone the repository

```bash
git clone https://github.com/cesmii/I3X-Explorer.git /path/to/i3X-Explorer
cd /path/to/i3X-Explorer
```

### 2. Configure nginx

Copy the example config and edit it for your server:

```bash
sudo cp scripts/nginx-web.conf.example /etc/nginx/sites-available/i3x-explorer
sudo nano /etc/nginx/sites-available/i3x-explorer
```

Update these values:
- `server_name` — your hostname or IP
- `root` — absolute path to `dist-web/` inside the repo
- `proxy_pass` — address and port of your I3X API server

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/i3x-explorer /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3. Fix directory permissions

nginx runs as `www-data` and needs read access to files inside the repo. Run once:

```bash
chmod o+x /home/YOUR_USER
chmod -R o+rX /path/to/i3X-Explorer/dist-web
```

### 4. Allow nginx reload without a password

The deploy script reloads nginx after each build. Grant the deploy user permission to do this without a password prompt:

```bash
echo 'YOUR_USER ALL=(ALL) NOPASSWD: /usr/bin/systemctl reload nginx' \
  | sudo tee /etc/sudoers.d/i3x-explorer
```

### 5. Install the systemd service

Copy the example service file and edit it for your server:

```bash
sudo cp scripts/i3x-explorer-web.service.example \
        /etc/systemd/system/i3x-explorer-web.service
sudo nano /etc/systemd/system/i3x-explorer-web.service
```

Update `User` and both path values (`WorkingDirectory` and `ExecStart`) to match your setup.

Enable and start the service, which will run the first build:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now i3x-explorer-web
```

Check it completed successfully:

```bash
sudo systemctl status i3x-explorer-web
# should show: active (exited)
```

## Runtime Configuration

By default the connection dialog shows the compiled-in server list. You can override this without rebuilding by creating a `config.local.json` file in the repo root:

```bash
cat > /path/to/i3X-Explorer/config.local.json << 'EOF'
{
  "serverUrl": "http://your-host:8090/api",
  "recentUrls": ["http://your-host:8090/api"]
}
EOF
```

This file is gitignored and persists across deployments. The deploy script copies it into `dist-web/config.json` after every build. The values are applied for first-time visitors; returning users keep whatever they saved in their browser's localStorage.

## Updating

After any code change:

```bash
cd /path/to/i3X-Explorer
git pull
sudo systemctl restart i3x-explorer-web
```

The service runs `scripts/deploy-web.sh`, which:
1. Runs `npm ci` to sync dependencies
2. Runs `npm run build:web` to produce `dist-web/`
3. Copies `config.local.json` into `dist-web/config.json` if present
4. Reloads nginx

## File Reference

| File | Purpose |
|------|---------|
| `scripts/nginx-web.conf.example` | nginx config template — copy to `/etc/nginx/sites-available/` |
| `scripts/i3x-explorer-web.service.example` | systemd service template — copy to `/etc/systemd/system/` |
| `scripts/deploy-web.sh` | Build and deploy script, run by the systemd service |
| `config.local.json` | Server-local runtime config (gitignored, create manually) |
| `public/config.json` | Default runtime config shipped with the build (empty values) |
| `dist-web/` | Built static files served by nginx (gitignored, generated by build) |
